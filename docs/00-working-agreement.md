# 00. Working Agreement

Muc tieu cua project nay la vua lam vua hoc. Vi vay cach lam mac dinh phai la **huong dan tung buoc**, khong phai assistant tu code het.

## Nguyen tac bat buoc

1. Moi lan chi lam mot mieng nho.
2. Truoc khi code phai noi ro:
   - Dang lam phan nao.
   - Vi sao can lam.
   - File nao se tao/sua.
3. Neu co code moi, phai giai thich code do lam gi.
4. Sau moi buoc phai dua lenh test.
5. Doi user test xong moi sang buoc tiep, tru khi user yeu cau assistant tu lam.
6. Khong nhay qua nhieu layer cung luc.
7. Neu assistant da tu lam nhieu hon can thiet, phai tom tat lai ro rang nhung gi da thay doi.

## Thu tu hoc backend

1. Model database.
2. Migration.
3. Schema request/response.
4. Service xu ly nghiep vu.
5. Route API.
6. Test tren Swagger.
7. Bao ve route bang auth neu can.

## Cach giai thich

Moi khai niem can giai thich bang ngon ngu truc tiep:

- Model la bang database.
- Schema la hinh dang data vao/ra API.
- Service la logic xu ly.
- Route la endpoint HTTP.
- Migration la lich su thay doi database.
- Dependency la cach FastAPI tiem san object vao function.

## Viec can tranh

- Khong code ca mot feature lon trong mot lan.
- Khong tao file moi ma khong noi muc dich.
- Khong sua migration/database ma khong bao rui ro.
- Khong mac dinh user da biet FastAPI, SQLAlchemy, Alembic, JWT.
